# Shopify 実装可能カタログ（できることリスト）

「Shopifyで〇〇したい」に対し「**できる/できない**」「**どの手段で**」「**何が前提か**」を即答するための網羅リスト。
2視点で整理：**A) 実行手段別** / **B) 機能ドメイン別**。末尾に早見表・注意点・実装メモ。

## 凡例（前提条件タグ）
- 💬 チャット完結可（テーマ編集→push / Admin APIトークン操作）
- 🔑 Admin APIの追加スコープが必要な場合あり
- 🧩 アプリ導入(App Store/Shopify製含む)が前提
- 🛠 CLIでのアプリ/拡張“開発”が必要（コードを書く）
- ☁ 動かし続けるサーバー(ホスティング)が必要
- 💎 Shopify Plus等の上位プラン条件
- 👤 ブラウザ/管理画面での人の操作・承認が必要（認証/決済/ドメイン/公開承認 等）
- 🇯🇵 日本案件で特に重要

## 難易度ラベル（技術難易度＋作業量）
- ⭐ Lv1：数分・ノーコード/値変更
- ⭐⭐ Lv2：数十分・テーマJSON/単発API
- ⭐⭐⭐ Lv3：半日〜数日・Liquid自作/設計
- ⭐⭐⭐⭐ Lv4：数日〜週・CLI開発/拡張/承認
- ⭐⭐⭐⭐⭐ Lv5：週〜月・要サーバー/運用/審査
- ⚠工数注意：難易度が低くても**件数が多いと工数は増える**（例：商品100点も、1点ずつ手動なら大／CSV・APIなら小）

# 難易度・工数別の早見（Lv1→Lv5）

## ⭐ Lv1：設定変更（数分・ノーコード・💬）
- 配色/フォント/余白・画像比率・ロゴ・ファビコン
- 商品のタグ/価格/在庫設定・商品の公開
- コレクション作成(自動分類)・並び順
- 文言変更・アナウンスバー・メニューにリンク追加

## ⭐⭐ Lv2：軽い実装（数十分・テーマJSON/単発API・💬）
- セクションの追加/並べ替え・ヒーロー設定・カテゴリー一覧
- メタフィールド＋**動的ソース**で項目表示（素材・お手入れ等）
- 商品CSV一括登録・画像アップロード(API)・コレクション公開
- URLリダイレクト・固定ページ/ポリシー作成・SEO(タイトル/メタ/OGP)

## ⭐⭐⭐ Lv3：中程度（半日〜数日・Liquid自作/設計・💬一部🛠）
- **独自セクション/ブロックの自作**（schema付き）
- 商品ページ/絞り込み(ファセット)/検索の作り込み
- メタオブジェクト設計・多言語(Markets)整備・メール文面編集
- ブランド世界観に沿った全体デザイン改修・代替テンプレート

## ⭐⭐⭐⭐ Lv4：重い（数日〜週・CLI開発/承認・🛠💎）
- **Shopify Functions**（割引/送料/決済/検証/ルーティング）
- テーマアプリ拡張(App Block)・チェックアウトUI拡張(💎)
- Web Pixels(計測)・Shopify Flow設計・B2B構築・Markets本格構築
- 既存サイトからの中〜大規模移行

## ⭐⭐⭐⭐⭐ Lv5：大規模（週〜月・要サーバー/運用/審査・☁👤）
- 埋め込みカスタムアプリ（独自業務ツール/外部連携/独自ダッシュボード）
- ヘッドレス(Hydrogen)でフロント全面構築
- App Store公開アプリ・ERP/基幹システム連携

### 工数を膨らませる主因
商品/画像/翻訳の**件数**、デザインの作り込み度、承認の往復回数、検証範囲(端末/ブラウザ)、外部連携の数。
→ 同じ機能でも「量」と「品質要求」で工数は大きく変わる。

# A. 実行手段別：それぞれで何ができるか

## A-1. テーマ編集 → `shopify theme push`（💬）
お店の「見た目・構成・体験」を担当。ファイル編集→下書きテーマへ反映→プレビュー→承認後に公開。
- `templates/*.json`：セクションの追加/並べ替え/削除・各セクション設定値・代替テンプレート(商品/コレクション別デザイン)
- `config/settings_data.json`：配色・フォント・全体デザイン設定の値
- `config/settings_schema.json`：テーマ設定“項目そのもの”の定義
- `sections/ blocks/ snippets/ layout/*.liquid`：部品のHTML/Liquidロジック/構造（独自セクション・ブロック・スニペット自作）
- `assets/*.css /*.js`：スタイル・スクリプト（AJAXカート等の挙動も）
- `locales/*.json`：多言語の文言
- `templates/robots.txt.liquid`：クローラ制御の編集
- セクショングループ(header/footer)、password/gift_cardテンプレート、アプリ埋め込みブロックの配置
- **動的ソース**：メタフィールド/メタオブジェクトをテーマ設定に“接続”して表示（後述・ノーコード寄り）

## A-2. Admin API（トークン, GraphQL）（💬🔑）
お店の「データ」を担当。`https://<shop>.myshopify.com/admin/api/<version>/graphql.json`（ヘッダ `X-Shopify-Access-Token`）。
- 商品・バリエーション・在庫・価格・タグ・**メタフィールド/メタオブジェクト(定義含む)**
- コレクション(自動/手動)・並び順・**公開(publishablePublish)**
- ファイル(画像)アップロード(fileCreate)→テーマは `shopify://shop_images/<filename>` で参照
- 顧客・注文・**ドラフト注文**・割引・**URLリダイレクト**・**販売プラン(selling plans)**
- **一括処理(Bulk Operations)** で大量データの取得/更新
- **Webhook購読**の登録（イベント→外部通知）
- スコープは必要分だけ（products/inventory/files/publications/content/metaobjects/discounts/orders/customers/translations/markets…）

## A-3. Shopify CLI：テーマ系（`shopify theme ...`）（💬一部👤）
- `dev`(ローカルプレビュー) / `push` / `pull`(取得) / `check`(品質)
- `list` / `rename` / `delete` / `share`(共有プレビューURL) / `package`(zip納品) / `console`(Liquidデバッグ) / `publish`(本番公開:要承認👤)

## A-4. Shopify CLI：アプリ/拡張“開発”（`shopify app ...`）（🛠 ほか）
「Shopifyに無い独自機能」の新規開発。`npm init @shopify/app` → `app dev` / `app deploy` / `app generate extension`。
- **Shopify Functions**(💚サーバー不要)：割引/送料/決済方法/カート変換/フルフィルメント制約/注文ルーティング/受取・配送オプション生成/**カート・チェックアウト検証**
- **チェックアウトUI拡張**(💎) / **管理画面UI拡張** / **顧客アカウントUI拡張** / **POS UI拡張**
- **テーマアプリ拡張(App Block / App Embed)**：テーマを汚さず差し込める部品・全体埋め込み(チャット/Cookie同意等)
- **埋め込み型カスタムアプリ**(☁要ホスティング) / **Webhook連携** / **Web Pixel拡張**(計測)

## A-5. ヘッドレス：Hydrogen + Oxygen（⚛☁🛠）
Storefront APIでフロントをReactで丸ごと自作。表示・体験を完全自由化(高速化/独自UI/独自CMS連携)。Oxygen(Shopifyのホスティング)等で配信。

# C. 判断の早見表（何で実現する？）
- 見た目/レイアウト/文言/独自セクション/動的ソース表示 → **A-1 テーマ編集** 💬
- 商品/在庫/コレクション/画像/公開/メタフィールド/割引/リダイレクト/一括処理 → **A-2 Admin API** 💬🔑
- テーマの取得/共有/納品/デバッグ/公開 → **A-3 CLIテーマ系**
- 「Shopifyに無い独自ロジック/独自UI/外部連携」 → **A-4 CLIアプリ/拡張(Functions等)** 🛠
- フロントを完全自作 → **A-5 Hydrogen** ⚛☁
- まず**設定/アプリで足りないか**を確認 → ダメなら**テーマ自作** → さらにダメなら**アプリ/Functions**、の順で検討

# D. 必ず人手/外部が要ること（チャットだけで完結しない）
- ブラウザ認証・アプリのインストール承認 👤
- 決済の有効化・独自ドメイン・プラン契約・本番公開承認 👤
- App Storeでの公開(審査・人と時間) 👤
- アプリ/ヘッドレスを動かす外部サーバー(ホスティング) ☁

# E. 実装メモ（要点）
- テーマ変更は**下書きテーマへpush→プレビュー確認→承認後に公開**(ライブ直編集しない)
- Admin APIは**最小スコープ**。トークンは**Git管理外**で保管、納品時はローテーション/無効化
- 画像は **fileCreate(originalSourceにURL, filename指定)** でFilesへ→ `shopify://shop_images/<filename>` で参照。Burst原寸は25MP超で弾かれるので `?width=2048〜2400`
- スマートコレクションは**オンラインストアへ公開(publishablePublish)**しないとストアフロントに出ない
- メタフィールド/メタオブジェクトは**動的ソース**でテーマにノーコード接続できる(コード無しで動的表示)
- 編集後は **`shopify theme check`** ＋ レスポンシブ/アクセシビリティ/表示速度の確認

# F. 日本案件の必須 🇯🇵
- **特定商取引法に基づく表記**(事業者名/住所/連絡先/返品条件 等) は必須
- プライバシーポリシー/利用規約/返品・配送ポリシー、会社情報
- Cookie同意/個人情報の扱い、適格請求書(インボイス)対応の検討
- 決済(コンビニ/代引き等 日本向け)・送料の地域設定
